### 2.2 核心组件

#### （1）意图识别引擎 (`parseIntent`)

基于正则表达式 + 关键词匹配，支持以下类型的自动识别：

| 任务类型 | 触发词 | 示例 |
|----------|--------|------|
| `daily_report` | 日报、今天、昨日、daily | "今天数据怎么样" |
| `weekly_report` | 周报、本周、上周、weekly | "生成本周运营周报" |
| `profit_diagnosis` | 利润、亏钱、ROI、毛利、成本 | "哪些产品在亏钱" |
| `influencer_report` | 达人、BD汇报、KOL、博主 | "达人BD周报" |
| `data_analysis` | 分析、数据、Excel、文件 | "分析这份GMV数据" |
| `knowledge_qa` | 默认兜底 | "马来西亚佣金多少" |

同时自动提取日期范围（今天/本周/上月等）、关注焦点（GMV/利润/达人/素材/广告）。

#### （2）数据采集模块 (`gather*`)

直接从 SQLite 数据库实时查询，不依赖缓存：

- `gatherOrderSummary()` — 按日期范围/店铺查询订单汇总（GMV、订单量、税费、运费、按状态/店铺分组）
- `gatherProductTop()` — Top N 热销产品排行（按销售额排序）
- `gatherProfitOverview()` — 利润概览（关联 financial_records 表，计算总利润/ROI/投资额）
- `gatherInfluencerSummary()` — 达人统计（总数、新增、寄样、Top 10）

#### （3）知识库 RAG 检索 (`searchKB`)

- 源数据：`docs/knowledge-base/` 下 **60+ 篇** Markdown 文档（10 大类）
- 检索方式：关键词分词 + 多级权重（标题×5、关键词×3、正文×1）
- 缓存策略：60 秒 TTL 自动刷新
- 返回格式：包含文件路径、标题、内容摘要、匹配分数

#### （4）多渠道 LLM 代理 (复用 `ai.ts`)

- 支持 8 个 AI 提供商：DeepSeek、豆包、硅基流动、OpenAI、Claude、Gemini、智谱、文心
- 负载均衡：按优先级依次尝试，失败自动切换下一个
- 统计追踪：成功/失败次数、平均延迟、最后错误信息
- 对 Skill 透明：无需在 Skill 层关心用哪个模型

#### （5）业务规则引擎（硬编码）

以下规则直接在代码中实现，不依赖 LLM 推理，确保计算精确：

| 规则 | 生效日期 | 值 |
|------|----------|-----|
| 交易手续费 | 始终 | 3.78% |
| BXP 费用 | 始终 | 4.86% |
| TikTok 佣金 | 2026/07/25起 | 10.26% |
| 平台支持费 | 2026/06/05起 | 0.45 MYR/单 |
| 跨境运费公式 | 始终 | 重量×0.015×汇率 |
| 汇率 | 每日9:00 | exchangerate-api.com |

### 2.3 Prompt 设计

每个任务类型有独立的系统提示词，存储在 `server/src/routes/skiis-chat.ts` 的 `SYSTEM_PROMPTS` 对象中。所有 Prompt 包含：

1. **角色定义** — 明确 AI 身份
2. **输出格式** — Markdown 模板
3. **业务规则** — 嵌入已知规则（对 LLM 陈述）
4. **质量要求** — 格式化、专业性、行动导向

## 三、应用场景

### 场景 1：运营日报自动化

**传统方式**：运营人员打开ERP → 查看订单列表 → 手动计算GMV/订单量 → 打开Excel做表 → 查看各店铺数据 → 手动写日报 → 30分钟

**工作体方式**：说"生成本周运营日报" → 3秒自动生成结构化Markdown日报

### 场景 2：利润异常诊断

**传统方式**：财务打开核算模块 → 逐条查看利润率 → 手动排查亏损产品 → 猜测亏损原因 → 查费率文档 → 20分钟

**工作体方式**：说"哪些产品在亏钱" → 自动识别Top5亏损产品 + 逐项拆解成本 + 给出优化建议

### 场景 3：达人BD周报

**传统方式**：BD打开达人模块 → 统计寄样数据 → 查跟进记录 → 写周报 → 15分钟

**工作体方式**：说"达人BD周报" → 自动统计达人合作数据 + 生成结构化汇报

### 场景 4：跨境电商知识查询

**传统方式**：搜索引擎 → 翻文档 → 找政策原文 → 不确定答案是否最新 → 10分钟

**工作体方式**：说"马来西亚佣金现在是多少" → 从60+篇知识库检索 + LLM解答 + 标注来源

### 场景 5：新员工上岗速成

**传统方式**：培训3天 → 熟悉ERP → 记住费率和规则 → 试错学习 → 1周

**工作体方式**：新员工直接问自然语言 → 系统自动应用正确的业务规则 → 即时产出合格结果

## 六、运行要求

- Node.js 22.12+ 
- 虾掌柜ERP系统已启动（`server/src/index.ts`）
- 至少配置 1 个 AI 渠道（DeepSeek/豆包/OpenAI等）
- 可选：`docs/knowledge-base/` 目录有知识库文件

## 七、扩展性

1. **新增任务类型**：在 `parseIntent` 添加关键词匹配 + 在 `SYSTEM_PROMPTS` 添加对应 Prompt
2. **新增数据源**：在 `gather*` 函数中添加新的数据库查询
3. **新增分析框架**：在 `SYSTEM_PROMPTS.data_analysis` Prompt 中描述新框架
4. **新增 AI 提供商**：通过 AI 渠道管理 UI 添加，无需改代码
5. **共享复用**：Skill 文件打包为 zip，团队成员导入即可使用

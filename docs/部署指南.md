# 虾掌柜ERP 服务器部署指南

## 准备工作

| 项目 | 要求 |
|------|------|
| 服务器系统 | Ubuntu 22.04 / CentOS 7+ / Alibaba Cloud Linux 3 |
| 最低配置 | 2核CPU + 2GB内存 + 40GB硬盘 |
| 推荐配置 | 2核CPU + 4GB内存 + 60GB SSD |
| 带宽 | 3Mbps 起步，推荐 5Mbps |
| 域名 | 可选（建议绑定，方便申请SSL证书） |

---

## 部署步骤（3 种方式任选）

### 方式一：一键脚本（推荐）

```bash
# 1. 把整个项目传到服务器
scp -r f:/tiktok-crm root@你的服务器IP:/opt/tiktok-crm

# 2. SSH 登录服务器
ssh root@你的服务器IP

# 3. 进入目录，执行部署
cd /opt/tiktok-crm
bash scripts/deploy.sh
```

### 方式二：Git 拉取

```bash
# 1. 先把项目推到 GitHub/Gitee 私有仓库

# 2. 服务器上拉取
cd /opt
git clone 你的仓库地址 tiktok-crm
cd tiktok-crm
bash scripts/deploy.sh
```

### 方式三：手动逐步执行

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | bash
systemctl enable docker && systemctl start docker

# 2. 创建 .env
cat > server/.env << 'EOF'
PORT=3000
DB_PATH=data/erp.db
UPLOAD_DIR=/app/data/uploads
EOF

# 3. 构建 + 启动
docker compose up -d --build
```

## 验证部署

```bash
# 查看容器状态（两个都是 Up 就对了）
docker compose ps

# 预期输出：
# NAME              STATUS
# bozone-server     Up xx seconds
# bozone-client     Up xx seconds

# 浏览器访问
http://你的服务器IP
```

## 绑定域名 + HTTPS（强烈建议）

```bash
# 1. 先把域名 A 记录解析到服务器 IP

# 2. 安装 certbot
apt install -y certbot python3-certbot-nginx

# 3. 修改 client/nginx.conf 中 server_name
#    server_name _;  →  server_name 你的域名.com;

# 4. 重新构建前端
docker compose up -d --build client

# 5. 申请 SSL 证书
certbot --nginx -d 你的域名.com

# 6. 证书自动续期（已自动配置，检查一下）
certbot renew --dry-run
```

## 日常运维

```bash
# 查看实时日志
docker compose logs -f

# 只看后端日志
docker compose logs -f server

# 重启
docker compose restart

# 更新代码后重新部署
git pull
docker compose up -d --build

# 完全停止
docker compose down
```

### 数据备份（非常重要！）

```bash
# 定时备份脚本（加到 crontab）
# 每天凌晨3点备份，保留最近7天

cat > /opt/backup.sh << 'SCRIPT'
#!/bin/bash
cd /opt/tiktok-crm
tar -czf /opt/backups/erp-backup-$(date +%Y%m%d).tar.gz server/data/
find /opt/backups/ -name "*.tar.gz" -mtime +7 -delete
SCRIPT

chmod +x /opt/backup.sh
mkdir -p /opt/backups

# 加入定时任务
(crontab -l 2>/dev/null; echo "0 3 * * * /opt/backup.sh") | crontab -
```

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| 80端口被占用 | `systemctl stop nginx` 停掉系统自带Nginx |
| 容器启动失败 | `docker compose logs` 查看报错信息 |
| 502 Bad Gateway | 后端没起来，`docker compose logs server` 排查 |
| 登录后跳回登录页 | 清除浏览器缓存，检查 `.env` 中 FRONTEND_URL |
| 上传文件失败 | 检查 `server/data/uploads/` 目录权限 |
| 首次登录密码 | `admin` 账号，密码为预置密码（详见数据库seed） |

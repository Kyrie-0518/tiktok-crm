## 三、详细功能设计

### 3.1 功能一：日报智能生成 【流程型 + 知识型】

**用户输入示例**：
- "生成本周运营日报"
- "生成7月1日的日报，重点关注GMV和素材数据"
- "对比今天和昨天的数据，生成日报"

**执行流程**：
1. 解析日期范围 → 调用订单API获取GMV/订单量
2. 调用店铺API获取各店铺数据
3. 调用达人API获取当天达人跟进/素材发布
4. 调用财务API获取当天利润概览
5. RAG检索知识库中相关分析框架
6. 调用LLM生成结构化日报（Markdown格式）
7. 存入 skiis_daily_logs 表
8. 输出日报内容

**输出模板**：
```markdown
# 📊 虾掌柜运营日报 — 2026年7月1日

## 核心指标
| 指标 | 今日 | 昨日 | 环比 |
|------|------|------|------|
| GMV (MYR) | 12,580 | 10,320 | +21.9% |
| 订单量 | 48 | 39 | +23.1% |
| 利润率 | 18.2% | 15.7% | +2.5pp |

## 各店铺表现
[按店铺维度的GMV/订单/利润率表格]

## 达人素材发布
[今日发布的素材数量、达人名称、链接]

## 异常预警
[利润异常订单、发货超时订单等]
```

### 3.2 功能二：数据文件智能分析 【执行型 + 知识型】

**用户输入示例**：
- "分析这份GMV数据，重点关注趋势和异常"
- "上传的素材数据，用漏斗分析框架解读"
- "对比这两个月的广告数据，ROI变化原因是什么"

**执行流程**：
1. 接收用户上传的 Excel/CSV 文件
2. 自动识别数据结构（列名中英文映射）
3. 匹配用户偏好分析框架（漏斗/人货场/AARRR/RFM/同期群/归因）
4. 执行分析计算（聚合、对比、趋势）
5. RAG检索知识库找到分析方法论
6. 调用LLM生成分析报告
7. 存入 skiis_data_files 表
8. 输出分析结论 + 关键指标表格 + 行动建议

### 3.3 功能三：利润智能诊断 【知识型】

**用户输入示例**：
- "本周哪个产品毛利最低，为什么"
- "分析订单 #1024 为什么不赚钱"
- "如果汇率跌到1.50，哪些产品会亏本"

**执行流程**：
1. 调用财务API获取产品/订单利润明细
2. 运行公式引擎（毛利率、保本ROI、净利润、整体ROI）
3. 关联成本项逐项拆解
4. 结合RAG知识库的利润分析方法论
5. 调用LLM生成诊断结论
6. 输出：亏损Top5产品 + 原因拆解 + 优化建议

### 3.4 功能四：达人BD汇报生成 【流程型】

**用户输入示例**：
- "生成本周达人BD周报"
- "汇总本月达人合作数据"
- "哪些达人ROI最高，推荐继续合作"

**执行流程**：
1. 调用达人API获取本周跟进记录
2. 统计寄样、素材发布、带货数据
3. 按店铺维度汇总
4. 关联达人-产品绑定关系
5. 调用LLM生成结构化周报
6. 输出达人汇报表

### 3.5 功能五：跨境知识问答 【知识型】

**用户输入示例**：
- "马来西亚TikTok佣金费率现在是多少"
- "Shopee和Lazada的物流费用对比"
- "斋月选品有什么注意事项"
- "1688采购后怎么走跨境物流最便宜"

**执行流程**：
1. RAG检索60+篇知识库（关键词+语义匹配）
2. 调用LLM基于检索结果生成回答
3. 标注引用来源（哪篇文档）
4. 如果知识库无答案，回退到LLM通用知识

| 业务规则准确性 | ❌ 幻觉风险 | ✅ 硬编码 + 日期判断 |
